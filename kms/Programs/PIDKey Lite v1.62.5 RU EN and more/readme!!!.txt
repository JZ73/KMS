



RU: 
 Переход на новую версию программы после обновления версии. 
 Для перехода на новую версию необходимо: 
 - распаковать новую версию из скачанного архива в папку с новым именем; 
 - переименовать файл базы pkbase.db, идущий в комплекте с новой версией, 
   например, pkbase_NEW.db (эта база возможно потребуется для добавления 
   из нее блокированных и неопределенных ключей); 
 - скопировать файл своей базы pkbase.db из папки старой версии программы 
   в папку новой версии программы; 
 - запустить новую версию программы (с вашей базой) и сразу же произвести
   ремонт базы (Crtl-R) для внесения возможных изменений в структуру базы. 

PIDKey Lite /unpack - распаковывает xrm-ms рядом с программой.
PIDKey Lite "c:\mykeysbase.db" - подключает свою базу с ключами.


 EN: 
 The transition to the new version after the version upgrade. 
 To switch to the new version you must: 
 - unpack the new version from the downloaded archive into a folder with
   a new name; 
 - rename database pkbase.db file bundled with the new version, for example,
   pkbase_NEW.db (this database may need to add it from the blocked and uncertain keys); 
 - copy the file to its base pkbase.db from the old version of the software
   folder to the new version of the program; 
 - to launch a new version of the program (with your base) and immediately 
   repair the database (Crtl-R) for possible changes to the database structure.
